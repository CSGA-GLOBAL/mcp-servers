Frequently Asked Questions

Common questions about COBOL Bridge integration.

What mainframe systems are supported?‚ñ∏
COBOL Bridge supports all major mainframe platforms: IBM z/OS, VSE, i (iSeries), and AIX. CICS, IMS, and batch job (JCL) processing are fully supported. Security models include RACF, ACF2, TopSecret, and custom authentication.
How does the CICS Bridge work?‚ñ∏
CICS Bridge creates a gateway between CICS Transaction Server (CTS) and modern REST APIs via CTG (CICS Transaction Gateway) or CICS web services. Real-time governance checks are executed at transaction entry points with minimal latency (50-200ms).
Is this CASA certified?‚ñ∏
Yes. COBOL Bridge includes CASA-CA30 certification for Defence/Enterprise deployments. Mission Critical tier (Enterprise) includes CASA-ISS certification for mainframe systems integration.
What security models are supported?‚ñ∏
COBOL Bridge integrates with RACF (Resource Access Control Facility), ACF2, TopSecret, and custom mainframe authentication systems. All data transmission uses TLS 1.3 with AES-256 encryption at rest.
How is latency handled in real-time governance?‚ñ∏
Real-time governance checks are optimized for 50-200ms latency via CTG connections. For critical transactions requiring sub-50ms response, batch processing with hourly/daily governance assessments is recommended.
